Minecraft Server Documentation

Deploy, configure and tune high‑performance Minecraft servers on Fexhub (Paper/Fabric/Forge).

Quick Start

  1. Go to Resources → Game Servers → Create.
  2. Template: Minecraft (Paper) (best performance) or Fabric/Forge for mods.
  3. Region: choose closest POP (see Network).
  4. Plan: start with 2–4GB RAM; scale later (see Pricing).
  5. Deploy — server is online in ~60 seconds.

Server Files & Folders

PathPurpose
/plugins/Paper/Spigot plugins (.jar)
/mods/Fabric/Forge mods
/world/Main world save
server.propertiesDifficulty, view‑distance, port

Performance Tuning

  • view-distance: 8–10 for survival servers.
  • simulation-distance: 6–10 depending on player count.
  • Use optimized Paper builds; remove heavy plugins; keep Java updated.

Backups & Snapshots

Create a Snapshot before mod/plugin changes. Restore from the panel if something breaks.

Common Tasks

Install a modpack

Stop server → upload ZIP to /mods → Start → watch console for missing libs.

Whitelist your server

In console: whitelist on, then whitelist add <player>.

Change MOTD

Edit server.properties → motd=Your Message → restart.

Troubleshooting

TPS drops/lag

Reduce render distance, remove heavy farms, upgrade to higher CPU plan.

Crashes on boot

Incompatible mods or wrong loader. Try a fresh Paper instance.
